# 8. Workflows de validation

[← Recherche](07-recherche.md) · [Retour au sommaire](README.md)

---

Un **workflow** est une suite d'**étapes ordonnées** que vos pages traversent — par
exemple un cycle éditorial *Brouillon → Revue → Publié*. Il structure la manière dont
un contenu est relu puis publié.

> La définition des workflows est réservée au **propriétaire** de l'espace.

---

## Créer un workflow

1. Ouvrez **Réglages de l'espace → Workflows**.
2. Cliquez sur **« Nouveau workflow »**.
3. Renseignez le **Nom du workflow**.
4. Définissez les **Étapes (dans l'ordre)**. Trois étapes sont proposées par défaut :
   **Brouillon**, **Revue**, **Publié** (cette dernière marquée *finale*).
   Pour chaque étape :
   - un **nom** ;
   - une case **« finale »** — marque une étape **terminale** (qui publie la page) ;
   - une icône **✕** pour la retirer.
5. **« + Ajouter une étape »** ajoute une étape supplémentaire.
6. Cliquez sur **« Créer le workflow »**.

> Les étapes sans nom sont ignorées à l'enregistrement ; l'ordre est conservé.

---

## Liste et suppression

L'onglet Workflows liste les workflows existants. Chaque ligne montre le nom et les
étapes reliées par des flèches, par exemple :

> **Brouillon → Revue → Publié**

Une icône **✕** (*Supprimer*) permet de retirer un workflow. S'il n'y en a aucun :
*Aucun workflow défini.*

---

## Faire avancer une page dans un workflow

Sur une page à laquelle un workflow s'applique, la barre d'actions affiche un **badge**
avec l'étape courante (icône de cycle).

- Un **éditeur** peut cliquer sur **« Faire avancer »** pour passer aux **étapes intermédiaires**.
- L'**étape finale publie** la page — **seul le propriétaire** peut la franchir.

Chaque changement d'étape génère une **notification** de type *workflow* auprès des
membres concernés (voir [Notifications](09-notifications.md)) : c'est ainsi que
l'équipe apprend qu'une page a progressé.

---

## Exemple de mise en place

1. Propriétaire : créez le workflow *Brouillon → Revue → Publié*.
2. Un éditeur rédige, puis clique **« Faire avancer »** → la page passe en *Revue*.
3. Un relecteur est notifié, vérifie le contenu.
4. Le propriétaire franchit l'étape **finale** → la page est **Publiée**.
